# Phase 5 — HTTP transport in `ui/`

**Status:** completed
**Owner:** agent (claude-code)
**Opened:** 2026-04-19
**Last update:** 2026-04-19

---

## Intent

Prove that `ui/` is not coupled to the CLI transport by adding a second transport —
HTTP — that reaches the same service layer. The architecture claim is that the
layered design survives a transport change without touching `service/`, `repo/`,
`runtime/wire.ts`, or any business logic. This phase tests that claim.

Secondary goal: make the HTTP entry point runnable (`npm run serve`) so that
integration evals can exercise the real network round-trip.

## Out of scope

- Middleware-like concerns (CORS, rate limiting, logging middleware) — defer to fastify migration or per-concern ADRs
- HTTPS / cert management — platform concern
- Authentication middleware (the CLI parses `actor` from flags; HTTP accepts an `X-Actor` header for now — auth is a future phase)
- OpenAPI / Swagger generation
- Websockets or streaming
- Fastify migration (see ADR 0004 for triggers)

## Approach

**Layer:**
- `src/ui/http/server.ts` — `buildHttpServer(services) → http.Server`. Binds nothing; caller calls `.listen()`.
- `src/ui/http/router.ts` — pure dispatch from `{method, path}` to a typed handler.
- `src/ui/http/validators.ts` — zod schemas for request bodies and path params.
- `src/ui/http/error-mapper.ts` — pure: `(err) → {status, body}`.
- `src/ui/http/main.ts` — entry point: `buildServices()` + `buildHttpServer()` + `.listen(PORT)` + SIGTERM handler.

**Actor passing:** HTTP clients send `X-Actor: user:<id>`. The router parses it into
`Actor` before calling the service. Same authz model as CLI, different source.

**Tests:** unit tests bind an ephemeral port (`.listen(0)`), grab the real port from
`server.address()`, and make requests via `fetch`. Server is torn down in `afterAll`.

**Evals:** a helper script starts `npm run serve` on a test port (via env var),
polls `/healthz` until ready, runs the scenario, kills the server. Skips cleanly if
`DATABASE_URL` is not set (like eval 0009).

## Risk & rollback

- **Risk:** port conflicts on a laptop with other services running. Mitigation: evals bind to port 0 (kernel picks); the `serve` entry point reads `PORT` env var with default 3000.
- **Risk:** the server process leaks on eval failure. Mitigation: eval helper uses `try/finally` to `kill()` the child; CI kills the job's processes anyway.
- **Risk:** the claim "zero changes to service/repo/runtime" quietly breaks during implementation. Mitigation: check `git diff --stat service/ repo/ runtime/wire.ts types/` before commit; if it's non-empty, stop and understand why.
- **Rollback:** `git revert` the Phase 5 commit. CLI still works.

## Decision log

### 2026-04-19 — Built-in http over fastify
- See ADR 0004.

### 2026-04-19 — Actor via `X-Actor` header, not auth middleware
- **Context:** we don't have auth yet; tests need to pass an actor.
- **Chose:** simple `X-Actor: user:<id>` header parsed by the router.
- **Because:** mirrors the CLI's `--actor=user:<id>` flag. Swap to real auth (JWT, session) in a future phase without changing the shape — the header-to-Actor mapping becomes an auth middleware output.
